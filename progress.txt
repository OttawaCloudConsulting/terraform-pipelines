# Source: prd.md + ARCHITECTURE_AND_DESIGN.md
# Spec: docs/ARCHITECTURE_AND_DESIGN.md is the authoritative design reference
# Variants: Default (9-stage, cross-account or single-account), Default-DevDestroy (10-11 stage)

## Features

[x] Feature 1: Core Module Extraction and Repository Restructure
    - Move root-level .tf files into modules/core/ and modules/default/
    - Move buildspecs/ to modules/core/buildspecs/
    - modules/core/ creates: 2 IAM roles + policies, S3 state bucket (conditional),
      S3 artifact bucket, SNS topic, CodeStar Connection (conditional),
      4 CloudWatch log groups, 4 CodeBuild projects
    - Core IAM policies accept optional additional_codebuild_project_arns and
      additional_log_group_arns inputs to support variant-owned resources
    - Core variables.tf: superset of all inputs needed by core resources
    - Core outputs.tf: all resource ARNs/names/IDs needed by variant wrappers
    - Core does NOT create CodePipeline (variants own that)
    - modules/default-dev-destroy/ empty skeleton exists
    - Restructure docs/ into nested format (shared/, default/, default-dev-destroy/)
    - Move diagrams to docs/shared/diagrams/
    - Repository root is clean (no .tf files except potentially a root versions.tf)
    - terraform validate passes for the core module in isolation
    NOTES: Started 2026-02-13. Completed 2026-02-13.

[x] Feature 2: Default Variant — Wrapper Module
    - Create modules/default/main.tf calling modules/core/ + defining 9-stage CodePipeline V2
    - 9 stages: Source > Pre-Build > Plan+Scan > Optional Review > Deploy DEV >
      Test DEV > Mandatory Approval > Deploy PROD > Test PROD
    - Uniform variable interface (all base variables, pass-through to core)
    - Uniform output interface (all 11 outputs, pass-through from core)
    - Include moved blocks so existing root module users migrate without resource destruction
    - Support single-account deployment when dev_account_id == prod_account_id
    - Functionally identical resources to the current monolithic module
    - terraform validate passes
    NOTES: Started 2026-02-13. Completed 2026-02-13.

[x] Feature 3: Default-DevDestroy Variant — Wrapper Module
    - Create modules/default-dev-destroy/ with 10-11 stage CodePipeline V2
    - Variant creates 5th CodeBuild project (destroy) + dedicated CloudWatch log group
    - Pass destroy CB project ARN and log group ARN to core via
      additional_codebuild_project_arns and additional_log_group_arns
    - enable_destroy_approval defaults to true (safe by default)
    - When true: manual approval stage before destroy (11 stages total)
    - When false: destroy runs automatically after PROD tests pass (10 stages total)
    - Destroy stage runs terraform destroy against DEV via cross-account role
    - Destroy buildspec at modules/default-dev-destroy/buildspecs/destroy.yml
    - Destroy CodeBuild project uses same CodeBuild service role from core
    - If destroy stage fails, pipeline shows as Failed and DEV remains intact
    - Uniform variable interface + enable_destroy_approval
    - Uniform output interface
    - terraform validate passes
    NOTES: Started 2026-02-13. Completed 2026-02-13. Destroy buildspec is placeholder (Feature 4).

[x] Feature 4: Destroy Buildspec
    - Create modules/default-dev-destroy/buildspecs/destroy.yml
    - Installs IaC runtime (terraform or opentofu)
    - IaC installation logic kept consistent with core buildspecs (deploy.yml)
    - Assumes cross-account DEV deployment role via sts:AssumeRole
    - Initializes Terraform with DEV state key
    - Runs terraform destroy -auto-approve
    - Handles var-file gracefully (checks if environments/dev.tfvars exists)
    - Uses set -euo pipefail
    NOTES: Started 2026-02-13. Completed 2026-02-13.

[x] Feature 8: IaC Working Directory Variable
    - New variable iac_working_directory (default: ".") in core, default, and default-dev-destroy
    - Allows Terraform/OpenTofu files to live in a subdirectory of the target repo
    - Passed as IAC_WORKING_DIR env var to plan, deploy, and destroy CodeBuild projects
    - Buildspecs (plan.yml, deploy.yml, destroy.yml) cd into IAC_WORKING_DIR before terraform commands
    - cicd/ scripts (prebuild, smoke tests) remain repo-root-relative — not affected by this variable
    - prebuild.yml and test.yml unchanged (no terraform commands, only cicd/ scripts)
    - Default value "." preserves existing behavior — fully backward compatible
    - terraform validate passes for both variants
    NOTES: Started 2026-02-16. Completed 2026-02-16.
